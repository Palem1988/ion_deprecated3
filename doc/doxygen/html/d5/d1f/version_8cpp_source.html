21 // The following part of the code determines the CLIENT_BUILD variable.
22 // Several mechanisms are used for this:
23 // * first, if HAVE_BUILD_INFO is defined, include build.h, a file that is
24 // generated by the build environment, possibly containing the output
25 // of git-describe in a macro called BUILD_DESC
26 // * secondly, if this is an exported version of the code, GIT_ARCHIVE will
27 // be defined (automatically using the export-subst git attribute), and
28 // GIT_COMMIT will contain the commit id.
29 // * then, three options exist for determining CLIENT_BUILD:
30 // * if BUILD_DESC is defined, use that literally (output of git-describe)
31 // * if not, but GIT_COMMIT is defined, use v[maj].[min].[rev].[build]-g[commit]
32 // * otherwise, use v[maj].[min].[rev].[build]-unk
